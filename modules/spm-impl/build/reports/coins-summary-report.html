Density is the number of rule violations per 1000 lines of code. It is colored red when there are 8.0 or more, amber for the range 1.0 to 8.0 and green when less than 1.0.

Priority the rule violations are separated into 5 priorities (only 3 are displayed). Priority 1 is the most serious and Priority 5 is the least serious.

Files is the number of files in the project containing rule violations.


The Delta columns (Δ1D, Δ7D, Δ28D) are the difference between the current value and those of the previous day, week or month. The difference is calculated as "current value - previous value". This means that negative values (green) indicate an improvement and positive values (red) indicate a decline. The previous value is the current percentage of the latest project, whose build time falls within one the following intervals: Δ1D - (24h - 1h, 3*24h + 1h), Δ7D - (7*24h - 1h, 10*24h), Δ28D - (28*24h - 1h, 42*24h). If none of the previous builds falls within one of these time intervals, N/A is displayed.
